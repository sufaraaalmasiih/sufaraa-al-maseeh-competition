# سفراء المسيح — Ambassadors of Christ

مسابقة كتابية تفاعلية **عربية (RTL)** تُدار مباشرةً من قِبل ميسّر: تجيب الفرق على هواتفها، وتُعرض المسابقة على شاشة جمهور (بروجكتر)، مع شاشة مدرب لمتابعة فريق واحد. **مسابقة واحدة حيّة في كل مرة** (تصميم مقصود).

---

## التقنيات

| الطبقة | الأداة |
|--------|--------|
| الإطار | Next.js 15 (App Router) · React 19 · TypeScript (strict) |
| البيانات | Firebase 12 — Auth (بريد/كلمة مرور) · Firestore (لحظي) · Storage |
| الواجهة | TailwindCSS · framer-motion · Radix UI |
| Excel | SheetJS (xlsx) — استيراد/تصدير بنك الأسئلة |
| الاستضافة | Vercel (نشر تلقائي عند الدفع إلى `main`) · Firebase Spark (بلا Cloud Functions) |

---

## التشغيل محلياً

```bash
npm install
npm run dev        # http://localhost:3000  (يستخدم WASM للـ SWC على ويندوز)
```

| الأمر | الغرض |
|-------|-------|
| `npm run dev` | خادم التطوير |
| `npm run typecheck` | فحص الأنواع (tsc --noEmit) — يجب أن يكون نظيفاً |
| `npx vitest run` | الاختبارات (وحدة + round-trip) |
| `npm run smoke` | اختبار دخان بعد النشر — يتأكّد أنّ `/api/admin/*` ترجع 401 لا 500 |
| `npm run firebase:deploy:rules` | نشر `firestore.rules` (يدوي) |

> **ملاحظة ويندوز:** البناء المحلي عبر `next build` قد يفشل (SWC الأصلي)؛ نشر Vercel على Linux يعمل بلا مشاكل.

## المراحل الأربع

1. **اجمعوا الكنوز** — أسئلة فردية سريعة (اختيار/ترتيب/فراغات).
2. **فتّشوا الكتب** — 4 مجالات، لاعب لكل مجال (توصيل/ترتيب آية/إكمال آية/صح‑خطأ) مع تصحيح آلي.
3. **على المحك** — لوحة Jeopardy مبنية على البنك (حتى 6 فئات)، نقاط بالصعوبة، صاحب الدور ثم سرقة.
4. **اثبتوا بالحق** — أسئلة جماعية بنقاط متتالية (streak).

سير اللعب: `waiting_players → competition_intro → stage1..4 → final_results → podium`.

---

## الأدوار والصلاحيات

- **`super_admin` (المشرف العام):** صلاحية كاملة — يبدأ المسابقات الرسمية، يدير المحتوى/الإعدادات، تبويب «الإدارة»، حذف الفرق، إدارة الطاقم.
- **`facilitator` (الميسّر):** يبدأ **حصص تدريب فقط** ويعدّل الأسئلة/الإعدادات **في التدريب فقط**؛ يدير المسابقة الرسمية مباشرةً دون تعديل محتواها.
- **`team` (الفريق):** يلعب على الهاتف. **`coach` (المدرب):** عرض فقط + اعتراضات. **الجمهور:** شاشة عامة بلا تسجيل دخول.

القاعدة المركزية: `features/facilitator/content-edit-permission.ts`.

## بنية المجلدات

```

tests/ + *.test.ts  اختبارات vitest
```

## ملاحظات تشغيلية

- أبقِ **نافذة الميسّر مفتوحة** أثناء المسابقة (تقود الأتمتة).
- **شغّل شاشة الجمهور كتبويب على نفس متصفّح الميسّر/المشرف المسجَّل دخوله** (ووصّله بالبروجكتر). على جهاز منفصل مجهول لا تظهر إجابات الإعلان (قيد قواعد Firestore). لا تُسجّل دخول فريق في متصفّح العرض.
- بعض الميزات الإدارية (حذف حساب فريق/مدرب أو تغيير كلمة مروره) تحتاج **`FIREBASE_SERVICE_ACCOUNT`** مضبوطاً على Vercel (Production) — وإلا تُحفظ بيانات Firestore فقط.
- بعد تعديل `firestore.rules` يجب نشرها يدوياً (Firebase Console → Rules → Publish).
